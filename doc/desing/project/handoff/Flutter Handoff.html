04 — Componentes → Widgets

Cada primitive del prototipo tiene un equivalente Flutter. Abajo el mapeo directo y un snippet de referencia para cada uno.

VetButton
Botón primario / secundario / danger / AI
Btn → ElevatedButton (primary) · OutlinedButton (secondary) · FilledButton.tonal (ai)
Altura: sm=32 · md=42 · lg=52. Radio: r2=8. Con icon leading opcional.
VetCard
Contenedor base con hairline
Card → Container con BoxDecoration (surface + border line + radius r3).
Evitar Material Card nativo — su sombra no está en sistema.
VetChip
Tag con tono (neutral/primary/urgent/ok/warn)
Chip → Container con bg = toneSoft, fg = tone, borderRadius: 999, size xs/sm.
VetField
Label + input + hint
Field + TextInput → Column (label AppText.label + TextFormField). El InputDecorationTheme global ya deja el look correcto.
PetAvatar
Avatar circular por especie
PetAvatar → CircleAvatar con emoji de especie (🐕/🐈/🐇/🐦) o imagen; ring urgent si urgent=true.
SectionTitle
Kicker + título + meta + trailing
SectionTitle → Row con Column(kicker+title) y Spacer+trailing.
VetToast
Notificación auto-dismiss
Toast → ScaffoldMessenger.showSnackBar con custom content (icono circular + texto). Duración 3s.
VetDialog / VetConfirm
Info / confirmación con ícono grande
InfoDialog, ConfirmDialog → showDialog + Dialog (no AlertDialog nativo). Ícono en círculo soft, botón full-width.
VetTabBar
Bottom navigation 5 ítems
TabBar → NavigationBar (M3) con blur. 5 destinos: Hoy / Pacientes / Consulta / Hospital / Perfil.

Ejemplo — VetButton

class VetButton extends StatelessWidget {
  final String label;
  final VoidCallback? onPressed;
  final VetButtonVariant variant;
  final VetButtonSize size;
  final IconData? icon;
  final bool full;

  const VetButton({
    super.key,
    required this.label,
    this.onPressed,
    this.variant = VetButtonVariant.primary,
    this.size = VetButtonSize.md,
    this.icon,
    this.full = false,
  });

  @override
  Widget build(BuildContext context) {
    final height = _height(size);
    final child = Row(
      mainAxisSize: MainAxisSize.min,
      children: [
        if (icon != null) ...[
          Icon(icon, size: 16),
          const SizedBox(width: 8),
        ],
        Text(label, style: AppText.button),
      ],
    );
    return SizedBox(
      height: height,
      width: full ? double.infinity : null,
      child: _byVariant(context, child),
    );
  }
}

Ejemplo — VetCard

class VetCard extends StatelessWidget {
  final Widget child;
  final EdgeInsets padding;
  final VoidCallback? onTap;

  const VetCard({
    super.key,
    required this.child,
    this.padding = const EdgeInsets.all(16),
    this.onTap,
  });

  @override
  Widget build(BuildContext context) {
    return InkWell(
      onTap: onTap,
      borderRadius: BorderRadius.circular(AppRadii.r3),
      child: Container(
        padding: padding,
        decoration: BoxDecoration(
          color: AppColors.surface,
          border: Border.all(color: AppColors.line),
          borderRadius: BorderRadius.circular(AppRadii.r3),
        ),
        child: child,
      ),
    );
  }
}

06 — Patrones recurrentes

SOAP Canvas — los 10 puntos obligatorios

La consulta es una sola pantalla con acordeones, no un wizard lineal. Cada sección tiene estado empty / partial / complete y contribuye al contador X/10. Firma bloqueada hasta 10/10.

Res. 8430 exige los 10 puntos, no exige orden. Permite completar en paralelo, saltar entre secciones, y dictar cualquiera con IA.

Micrófono focus-aware

Un solo FAB de micrófono persistente en la parte inferior de la consulta. Lee la sección activa (focusedSection) y rellena ese campo con el resumen estructurado. Nunca abre pantalla aparte.

Future<void> onMicTap() async {
  final section = ref.read(focusedSectionProvider);
  final audio = await aiService.record();
  final summary = await aiService.summarize(audio, section: section);
  ref.read(consultationProvider.notifier).applyAi(section, summary);
  // marca con ✦ para trazabilidad
}

Contenido generado por IA

Todo lo que venga del servicio IA se marca con isAiGenerated: true y renderiza con un dot azul (AppColors.ai) previo al texto. Es editable en el acto — el usuario puede corregir y la bandera cae a isAiGenerated: false.

Toasts por acción

Guardar/editar/eliminar siempre disparan toast. Nunca diálogo modal para confirmación positiva. Solo usar ConfirmDialog para acciones destructivas (alta, cerrar HC, borrar paciente).

Pantallas críticas (hospital)

Vitales fuera de rango → pintar la card con urgentSoft + texto urgent. Tareas vencidas → mismo patrón. Nunca usar solo color — siempre combinar con icono de alerta para accesibilidad.

07 — Arquitectura sugerida

State management

Riverpod 2.x. Providers por feature. Evita setState global — solo UI local.

Persistencia

Isar para datos locales (pacientes, HCs, internados). HC firmada es inmutable — crea un Addendum aparte, nunca edites el original. Hive está bien para settings/cache.

Sync

La app debe funcionar offline (consulta en campo). Cola de writes que se envían cuando hay red. Mostrar banner warn cuando hay pendientes.

Servicio de IA

abstract class AiService {
  Future<String> transcribe(File audio);
  Future<AiSummary> summarize(String text, {required SoapSection section});
}

class AiSummary {
  final String structured;  // texto limpio
  final List<String> tags;    // chips sugeridos
  final double confidence;
}

Navegación

GoRouter con StatefulShellRoute para mantener la NavigationBar persistente. Splash y Login quedan fuera del shell.

Próximos pasos sugeridos

  1. Crear proyecto Flutter + copiar lib/theme/.
  2. Implementar las 6 primitives en lib/widgets/ (Button/Card/Chip/Field/PetAvatar/SectionTitle).
  3. Empezar por Splash → Login → Dashboard (flujo de arranque).
  4. Construir el SOAP Canvas de consulta — es la pieza central, le dedicaría 2 sprints.
  5. Integrar servicio IA (mock primero, real después).
  6. Hospitalización + Historia — son lectura con sheets, rápido.
